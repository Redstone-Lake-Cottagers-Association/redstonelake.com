# Membership & event form delivery setup

One Google Sheet + Apps Script webhook handles BOTH website forms:

- /membership → `/api/membership` → "Registrations" tab → emails membership@ + president@
- /events/submit → `/api/event-submission` → "Event submissions" tab → emails
  communications@ + president@
- Dev/preview submissions (test: true) go to "Test registrations" / "Test event
  submissions" tabs and email ONLY the webmaster — same rule for both forms.

## The sheet IS the events source (since July 2026)

The website reads its events calendar FROM this sheet (via the script's doGet):

- Each event row has a **Status** dropdown: `Pending` (new submissions), `Approved`,
  or `Rejected`. **Only Approved rows appear on the website** — flipping Status to
  Approved publishes an event within ~5 minutes (the site caches for 5 min).
- The production site reads the "Event submissions" tab; localhost/dev reads the
  "Test event submissions" tab (`?test=1`).
- Only public fields are exposed (title, date, time, location, type, description,
  details, icon, color) — submitter name/email/phone NEVER leave the sheet.
- Board members can add events directly as rows (set Status to Approved); the
  optional Icon (emoji) and Color (hex) columns customize the event card.
- `src/data/events.json` in the repo is only an emergency fallback if Google is
  unreachable — it is not the source of truth anymore.
- After first installing this script version, run `setupEventSheet` once from the
  Apps Script editor (function dropdown → setupEventSheet → Run) to add the
  Status/Icon/Color columns and dropdowns to both event tabs.

The registration form on /membership POSTs to `/api/membership`, which tries, in order:

1. **Google Sheet webhook** (`MEMBERSHIP_SHEET_WEBHOOK` Fly secret) — preferred. Each
   registration appends a row to a Google Sheet Donna can manage, and Google emails the
   coordinators. No AWS needed.
2. **AWS SES** (`AWS_ACCESS_KEY_ID` / `AWS_SECRET_ACCESS_KEY` / `AWS_REGION` /
   `MEMBERSHIP_FROM` secrets) — sends a plain email to membership@ and president@.
3. Neither configured → the form falls back to a pre-filled email the applicant sends
   themselves, so the flow never dead-ends.

## Google Sheet setup (~5 minutes, one time)

1. Create a Google Sheet named e.g. "RALA Membership Registrations" (use the association's
   Google account so Donna and the president have access).
2. Extensions → Apps Script, replace the contents with:

The full script lives in **docs/sheet-apps-script.js** — paste that entire file.

3. Deploy → New deployment → type **Web app** → "Execute as: **Me**", "Who has access:
   **Anyone**" → Deploy. Copy the web app URL (`https://script.google.com/macros/s/…/exec`).
4. `fly secrets set MEMBERSHIP_SHEET_WEBHOOK="<that URL>" -a redstonelake-com`
5. Test: submit the form on the site; a row should appear and both coordinators get an email.

Troubleshooting: an HTTP 403 "Access Denied" from the /exec URL means "Who has
access" is not set to Anyone — edit the deployment (Deploy → Manage deployments →
pencil icon) and change it; the URL stays the same.

## Updating the script later (keeps the same URL)

To change the script code (e.g. the event-handling update of July 2026): paste the new
code, then Deploy → **Manage deployments** → pencil icon on the existing deployment →
Version: **New version** → Deploy. The /exec URL stays the same, so no secret changes
are needed. (Only "New deployment" generates a new URL.)

Notes: the "Anyone" access is required for the site to POST; the URL is unguessable and the
script only appends rows. Donna can also add her own notification rules in the Sheet
(Tools → Notification settings).
